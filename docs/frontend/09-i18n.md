# Frontend — Internationalisation (i18n)

## Configuration (`src/i18n/i18n.ts`)

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import LanguageDetector from "i18next-browser-languagedetector";

i18n
  .use(LanguageDetector)       // Détecte la langue depuis: localStorage, navigator, querystring
  .use(initReactI18next)       // Intégration React
  .init({
    resources: {
      en: { translation: en },
      fr: { translation: fr },
      ar: { translation: ar },
    },
    fallbackLng: "en",         // Langue de secours
    interpolation: { escapeValue: false },  // React échappe déjà les strings
  });
```

## Langues supportées

| Code | Langue | Fichier |
|---|---|---|
| `en` | Anglais | `locales/en.json` |
| `fr` | Français | `locales/fr.json` |
| `ar` | Arabe | `locales/ar.json` |

## Usage dans les composants

```typescript
import { useTranslation } from "react-i18next";
// ou via le wrapper custom :
import { useTranslation } from "@/hooks/useTranslation";

const { t } = useTranslation();

// Clé simple
t("nav.dashboard")           // → "Dashboard" / "Tableau de bord"

// Avec variable
t("projects.confirm_delete_description", { name: project.name })
// → "Are you sure you want to delete 'My Project'?"

// Clé de colonne de table
t(`dataTable.columnNames.${column.id}`, column.id)
```

## Changement de langue

La langue est détectée automatiquement depuis `navigator.language` du navigateur.
Pour changer manuellement : `i18n.changeLanguage("fr")`.

**Note :** Il n'existe pas de sélecteur de langue visible dans l'UI actuelle. La détection automatique est la seule méthode.

## Support RTL (Arabe)

La langue arabe (`ar`) est incluse dans les fichiers de traduction mais il n'y a pas de gestion `dir="rtl"` sur la balise `<html>`. À ajouter pour un support complet.
